# 鐘聲系統（Python 3.12.8 / customtkinter）開發 TODO 清單

## 執行規則與重要概念
- **唯一依據**: 本清單為執行與回報的唯一依據，完成即打勾並填寫回報。
- **環境固定**: 使用 Python 3.12.8 與本專案虛擬環境 `.venv`；安裝/執行全部在該環境內完成。
- **執行緒模型**:
  - GUI 主緒（customtkinter）不可阻塞。
  - 排程與音訊播放置於背景緒；與 GUI 互動採 thread-safe 機制（queue 或 `after()` 回主緒）。
- **音訊後端策略**:
  - 預設 `pygame.mixer`：URL 音訊先下載到暫存檔再播放，結束後清理。
  - 可切換 `python-vlc` 以直接串流 URL（需已安裝 VLC）。
- **設定保存**:
  - 使用 JSON 檔存放：鬧鐘開關、播放清單、播放模式、後端選擇等。
  - 路徑策略：預設與程式同目錄；打包後支援可寫路徑（如 `%APPDATA%/SchoolBell`）。
- **日誌與偵錯**:
  - 使用 `logging`，滾動檔案（大小或天數輪替），等級 INFO/ERROR。
  - 重大事件（響鈴、播放失敗、重新連結）需有明確日誌。
- **錯誤處理**:
  - 網路下載設超時與重試（退避策略），失敗需 UI 提示與日誌。
  - 檔案不存在或權限錯誤需友善提示與重試入口（重新連結）。
- **打包與發佈**:
  - `PyInstaller` 6.8，`--onefile`；選擇性 `--noconsole`。
  - 在乾淨 Windows 測試（無 Python）驗證啟動、播放、存取權限。
- **回報格式（每步驟必填）**:
  - 完成情況：[ ]→[x]
  - 摘要：一句話結果
  - 產出：檔案/程式/截圖重點
  - 風險/阻礙：若有，含處置
